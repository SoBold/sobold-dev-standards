# Device Setup

This guide covers the initial configurations and tools needed to be able to develop at SoBold. Each section includes clear steps and guidelines to get everything running properly.

Additionally, each section will address both Mac and Windows workflows to ensure complete support for both operating systems.

<Callout type="info">
The setup guide follows a goal-oriented approach. By focusing on achieving the final objective, it ensures that all necessary tools are installed as needed throughout the process.
</Callout>

<Callout>
The setup guide should be followed sequentially. Each page assumes that the tools from the previous page have already been installed.
</Callout>

More info for each program is listed below.

## Overview

This setup guide is divided into five main sections:

1. **Code Editor Configuration**
2. **Version Control Setup**
3. **Node Dependency Management**
3. **PHP Dependency Management**
4. **Local Development Tools**

## Software Used

Here is all the software we use. Certain programs are responsible for different actions, so they will be listed in different subpages of *Setup*.

### <i class="devicon-homebrew-plain colored"></i> Homebrew
[Homebrew](https://brew.sh) is a package manager for macOS that simplifies the installation and management of software and utilities. It helps in installing various tools and applications through a command-line interface.

### <i class="devicon-nodejs-plain colored"></i> Node.js
[Node.js](https://nodejs.org) is a JavaScript runtime built on Chrome's V8 JavaScript engine, designed for building scalable network applications. It allows developers to execute JavaScript on the server side and is essential for running various JavaScript tools and libraries.

### <i class="devicon-npm-original-wordmark colored"></i> npm
[npm](https://www.npmjs.com) is a package manager for Node.js, used for managing JavaScript libraries and tools. It comes bundled with Node.js and is essential for installing and managing packages and dependencies.

### <i class="devicon-composer-plain "></i> Composer
[Composer](https://getcomposer.org) is a dependency manager for PHP, allowing developers to manage and install PHP libraries and tools. It simplifies the management of PHP dependencies and packages.

### <i class="devicon-php-plain colored"></i> PHP
[PHP](https://www.php.net) is a popular general-purpose scripting language that is especially suited for web development. It is widely used for creating dynamic web pages and server-side applications.

### <i class="devicon-fnm-plain colored"></i> fnm (Fast Node Manager)
[fnm](https://github.com/Schniz/fnm) is an alternative to nvm for managing multiple versions of Node.js. It is known for its speed and simplicity in handling Node.js versions.

### <i class="devicon-windows11-original colored"></i> Windows Package Manager (winget)
The [Windows Package Manager (winget)](https://docs.microsoft.com/en-us/windows/package-manager/) is a command-line tool for managing software on Windows. It simplifies the installation, updating, and management of applications through a unified interface.
